## 1. 현재 구조 분석

### 1.1 현재 파일 구조
```
react-app/src/
  App.jsx     ← 2277줄 (legacyMarkup + legacyScript + App 컴포넌트)
  main.jsx
  App.css
```

### 1.2 현재 동작 방식
```
App.jsx (React)
  ├── legacyMarkup: HTML 문자열 → dangerouslySetInnerHTML
  ├── legacyScript: JS 문자열 → document.createElement('script')
  └── useEffect: Firebase 로드 후 legacyScript 주입
```

**문제점**:
- 전체 앱 로직이 문자열로 관리 → IDE 지원 없음, 타입 체크 불가
- 전역 함수(`window.__roomAssignmentApi`) 의존
- HTML 이벤트 핸들러가 전역 스코프에 의존 (`onclick="saveStaff()"`)
- 2000줄 단일 파일 → 함수 찾기 어려움

## 2. 목표 구조 (Option C)

```
react-app/src/
  App.jsx                  ← ~150줄 (마크업 조립 + useEffect)
  main.jsx
  App.css
  lib/
    firebase.js            ← Firebase 초기화·연결·에러처리
    storage.js             ← localStorage 저장/불러오기·마이그레이션
    staff.js               ← 담당자 CRUD·검증
    assignment.js          ← 자동배정 알고리즘 (평일·토요일)
    schedule.js            ← 슬롯 계산·테이블 렌더링
    calendar.js            ← 캘린더 렌더링
    utils.js               ← 공통 유틸 (toMin, toTime, uid, pad2 등)
  components/
    markup.js              ← legacyMarkup HTML 문자열 (분리만)
```

> **점진적 전환 원칙**: `onclick="..."` 전역 함수 방식은 1차에서 유지. 로직만 모듈로 분리하여 각 파일이 독립적으로 테스트·수정 가능하게 만든다.

## 4. App.jsx 목표 구조

```jsx
// App.jsx (~150줄)
import { useEffect } from 'react'
import './App.css'
import { legacyMarkup } from './components/markup.js'

// legacyScript는 lib/ 모듈들을 import하여 조립
const legacyScript = `
  import { ... } from './lib/utils.js';
  // ... 모듈 조립 및 전역 함수 등록
`

function App() {
  useEffect(() => {
    // Firebase 로드 → legacyScript 주입 (기존 방식 유지)
  }, [])
  return <div dangerouslySetInnerHTML={{ __html: legacyMarkup }} />
}
```

> **참고**: legacyScript가 ES module import를 사용하려면 `type="module"` script 태그 필요. 이 부분은 구현 시 검토.

## 5. 전역 함수 관리 전략

현재 HTML에서 `onclick="saveStaff()"` 형태로 전역 함수를 호출함.

**1차 (이번 구현)**:
- `window.__roomAssignmentApi` 유지
- 각 전역 함수를 lib 모듈 함수로 위임하도록 래퍼만 변경
- HTML `onclick` 방식 그대로 유지

**2차 (미래)**:
- HTML 마크업을 JSX로 전환
- `onClick={handleSave}` 방식으로 교체
- 전역 함수 제거

## 7. 리스크 및 대응

| 리스크 | 대응 |
|--------|------|
| ES module import가 legacyScript 내에서 동작하지 않음 | IIFE 방식으로 모듈 export 없이 전역 변수 분리, 또는 Vite 번들링 활용 |
| `window.__roomAssignmentApi` 타이밍 문제 | 각 모듈 함수 독립 실행 가능하도록 순수 함수 설계 |
| 리팩토링 중 기존 기능 중단 | 각 모듈 분리 후 바로 앱 실행하여 확인 |
