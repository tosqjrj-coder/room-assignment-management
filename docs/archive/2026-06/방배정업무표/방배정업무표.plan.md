| **Problem** | 2000줄 단일 파일(App.jsx)에 HTML/JS 로직이 문자열로 내장되어 유지보수가 어렵고, 클라우드 저장/불러오기 메시지가 깨지는 버그가 존재함 |

## 1. 현재 상태 분석

### 1.1 앱 개요

- **기술 스택**: React + Vite, Firebase Firestore
- **파일 구조**: `react-app/src/App.jsx` (2277줄) — 사실상 전체 앱 로직이 단일 파일에 존재
- **동작 방식**: `legacyMarkup` (HTML 문자열) + `legacyScript` (JS 문자열)를 `useEffect`에서 DOM에 직접 주입

### 1.2 주요 기능

| 기능 | 설명 |
|------|------|
| 인증 | 비밀번호 입력 후 Firebase 자동 연결 |
| 담당자 관리 | 이름·근무구분·근무시간·활성화·자동배정 여부 설정 |
| 방 목록 | 1·2·3·4-1·4-2·5·6번방, 주니어, 헤드 (총 9개) |
| 평일 배정 | 화~금 10:00~21:00, 30분 단위, 오전/오후 shift + 홀짝 방 교대 자동 배정 |
| 토요일 배정 | 10:00~19:00, 13:00~14:00 공통 휴게, 블록별 인원 자동 배정 |
| 캘린더 | 날짜 선택 → 해당 날짜 클라우드 데이터 로드 |
| 클라우드 저장 | Firebase Firestore dailySchedules/{date} 문서로 저장 |
| 기본 담당자 | defaultStaffSettings 컬렉션에 저장/불러오기 |
| 자동배정 오버라이드 | 날짜별 autoAssignOverrides 설정 모달 |
| 포인트 합산 | 30분 배정 = 1P, 평일/토요일 분리 표시 |

### 1.3 데이터 구조

```js
// localStorage: STORAGE_KEY (roomScheduleAppV22)
{
  version: 22,
  activeMode: "weekday" | "saturday",
  staff: [{ id, name, worksWeekday, worksSaturday, isActive, staffAutoAssignDefault, workStart, workEnd, breakStart, breakEnd }],
  assignments: {
    weekday: { "slot_roomId": staffId },
    saturday: { "slot_roomId": staffId }
  },
  autoAssignOverrides: { [date]: { [staffId]: boolean } },
  settings: { startParity: "odd" | "even" }
}

// Firestore: dailySchedules/{date}  — 위와 동일 + updatedAt
// Firestore: defaultStaffSettings/default — { staffList, updatedAt }
```

## 2. 발견된 버그 목록

### 2.1 클라우드 메시지 인코딩 버그 (P1 — Critical)

**위치**: `App.jsx` 내 `loadFromCloud`, `saveDefaultStaffToCloud`, `loadDefaultStaffFromCloud` 함수

**증상**: 클라우드 관련 알림 메시지가 `???? ?? ? ???? ?????` 형태로 깨져 표시

**원인**: `legacyScript` 문자열 내부에서 한국어를 유니코드 이스케이프(`\uXXXX`)로 작성하거나, 템플릿 리터럴 경계에서 인코딩 손실 발생

**수정 대상 라인**: 1673, 1679, 1689~1706 (근사치)

### 2.2 전역 함수 접근 방식 (P2 — Structural)

**위치**: HTML 문자열 내 `onclick="saveStaff()"` 등 모든 이벤트 핸들러

**증상**: 현재는 동작하지만, 코드 분리 시 전역 스코프 의존성 때문에 이벤트 연결 실패 위험

## 3. 개선 범위 및 우선순위

### 3.1 Phase 1 — 버그 수정 (즉시)

| 항목 | 내용 |
|------|------|
| 깨진 메시지 수정 | `loadFromCloud`, `saveDefaultStaffToCloud`, `loadDefaultStaffFromCloud` 내 한국어 메시지 복원 |
| 인코딩 방식 통일 | 모든 한국어 문자열을 직접 문자열로 사용 (유니코드 이스케이프 제거) |

### 3.2 Phase 2 — 코드 구조 개선 (중기)

| 항목 | 방향 |
|------|------|
| 로직 모듈화 | `legacyScript` 내 함수들을 별도 `.js` 파일로 분리 (firebase.js, assignment.js, staff.js 등) |
| HTML 마크업 분리 | `legacyMarkup`을 React JSX 컴포넌트로 전환 |
| 전역 함수 제거 | `onclick="..."` 방식을 React 이벤트 핸들러로 교체 |
